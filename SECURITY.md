# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.3.x   | :white_check_mark: |
| 1.2.x   | :x:                |
| < 1.2   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in ERKLIG, please report it responsibly.

### How to Report

1. **DO NOT** open a public GitHub issue
2. Email: security@iamcanturk.dev
3. Or DM on Twitter: [@iamcanturk](https://twitter.com/iamcanturk)

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response Timeline

| Action | Timeframe |
|--------|-----------|
| Initial response | 24-48 hours |
| Vulnerability assessment | 3-5 days |
| Fix development | 1-2 weeks |
| Public disclosure | After fix release |

### Recognition

Security researchers who report valid vulnerabilities will be:
- Credited in our release notes
- Listed in our Hall of Fame
- Acknowledged on Twitter

## Security Best Practices

When using ERKLIG:

1. **Run with minimal privileges** - Don't run as root unless necessary
2. **Review before deletion** - Always check files before removing
3. **Backup first** - Backup your web directory before cleanup
4. **Update regularly** - Keep ERKLIG updated for latest signatures

## Disclaimer

ERKLIG is a detection tool, not a security guarantee. Always:
- Maintain regular backups
- Keep software updated
- Use multiple security layers
- Follow security best practices
